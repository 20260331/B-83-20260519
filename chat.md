## 修复内容

构建报错的原因是 5 个 `TS6133`（声明但未使用）TypeScript 严格错误：

1. **`Dashboard.vue:7`** — `Document`, `User`, `TrendCharts`, `Tickets` 四个图标组件在 `<script setup>` 中 import 了，但在模板中是通过 `<component :is="stat.icon" />` 动态引用字符串名称来使用的，TypeScript 编译器无法追踪到这种动态解析，判定为未使用。由于 `main.ts` 已全局注册了所有图标，移除这四个局部 import 即可，动态组件解析不受影响。

2. **`Dashboard.vue:186`** — `handleFileChange` 的 `file` 参数声明后未使用，加 `_` 前缀（`_file`）标记为有意忽略，这是 TypeScript 的惯用做法。
